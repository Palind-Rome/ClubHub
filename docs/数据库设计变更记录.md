## 2026-07-09 纳新创建人字段

### 变更对象

表：`RECRUITMENTS`

新增字段：

| 字段名 | 类型 | 可空 | 外键 | 说明 |
| ------ | ---- | ---- | ---- | ---- |
| `creator_user_id` | `number` | 是 | `USERS(user_id)` | 纳新创建人，用于草稿可见性、草稿删除权限、提交审核归属和“本社团提出的纳新”分组。 |

### 设计说明

- 草稿纳新只允许创建人查看、编辑和删除，因此需要在纳新记录上保存创建人。
- 同社团干部或负责人查看列表时，已提交审核或已通过的本社团纳新会被归入“本社团提出的纳新”；草稿仍只对创建人可见。
- 审核通过后的纳新不直接存储“未开始”“申请中”“已结束”，而是由后端根据 `start_at`、`end_at` 和审核通过状态自动计算展示状态。
