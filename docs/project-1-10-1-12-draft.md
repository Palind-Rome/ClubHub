# 1.10 与 1.12 项目协作模块开发草案

## 负责范围

- 1.10 社团项目立项申请与负责人分配。
- 1.12 项目阶段成果提交与审核。
- 本草案用于 Draft PR 占位与开发范围说明，后续实现以现有 `api/`、`backend/`、`database/` 结构为准。

## 计划接口方向

- 项目立项申请：创建项目，记录所属社团、项目名称、简介、负责人、起止时间和项目状态。
- 项目审核：审核项目立项申请，记录审核人和审核意见。
- 成果提交：基于项目任务提交阶段成果，记录成果标题、说明、文件地址和提交人。
- 成果审核：审核阶段成果，记录审核状态、审核人、审核意见和审核时间。

## 数据库关联

- `PROJECTS`：承载项目立项、负责人分配与项目审核状态。
- `PROJECT_TASKS`：承载任务信息以及阶段成果提交、审核信息。
- `USERS`：关联项目负责人、成果提交人和审核人。
- `CLUBS`、`CLUB_MEMBERS`：用于校验项目所属社团和成员身份。
