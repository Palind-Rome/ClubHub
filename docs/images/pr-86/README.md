# PR #86 截图说明

本目录用于保存 PR #86 的界面说明截图。截图基于本地预览服务和演示数据生成，只用于说明界面结构、入口位置和流程关系，不代表演示库中的真实业务数据。

## 截图清单

1. `01-club-award-tab.jpg`
   - 展示“评优评奖”入口已经下移到社团组织管理页内。
   - 它与“成员分组与任期”“学期考核”等页签并列，不再作为顶部全局导航入口。
   - 身份区域展示可参与评优评奖的能力标签，便于审查入口位置和角色上下文。

2. `02-award-workbench-lists.jpg`
   - 展示评优评奖独立工作台总览。
   - 上半部分是评优评奖活动列表、状态、申报数量和公示操作。
   - 下半部分是当前活动的成员申报记录、申报奖项、审核状态和处理入口。

3. `03-award-create-dialog.jpg`
   - 展示负责人发布评优评奖活动的弹窗。
   - 可设置社团、评奖标题、类型、学期和活动说明。
   - 该弹窗独立于学期考核录入，避免评奖配置混入成绩考核。

4. `04-award-apply-dialog.jpg`
   - 展示成员主动提交评优评奖申报的弹窗。
   - 成员围绕具体活动填写申报奖项和申报理由。
   - 该流程体现“成员申报”，不是负责人直接在成绩考核中录入奖项。

5. `05-award-review-dialog.jpg`
   - 展示负责人、干部或更高层级审核申报的弹窗。
   - 审核人可以选择通过或驳回，并填写审核意见。
   - 该截图对应后端的初审、终审和驳回状态流转。

6. `06-semester-evaluation-tab.jpg`
   - 展示“学期考核”页签保持独立。
   - 页面只保留成员、学期、活动分、任务分、学习分、荣誉加分、总分、等级和公示状态。
   - 不再承载评优评奖活动发布、成员申报或审核逻辑。

## 审查重点

- 入口位置：评优评奖在社团页内，与成员任期、学期考核并列。
- 数据边界：学期考核只做成绩/绩效评价，评优评奖作为独立流程。
- 流程完整性：活动发布、成员申报、层级审核、结果公示都有对应界面。
- 角色体验：负责人和成员看到的主要操作入口都能从截图中定位。
