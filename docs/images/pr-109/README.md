# PR #109 截图说明

本目录用于保存 PR #109 的界面说明截图。截图基于本地前端服务和当前演示数据生成，用于说明入口拆分、页面职责和角色可见内容，不代表演示库中的真实业务数据。

## 截图清单

1. `01-my-clubs-overview.png`
   - 展示原社团页已改名为“我的社团”。
   - 页面只保留社团基本信息和我的社团身份，不再承载成员管理、注册审核等重工作流。
2. `02-member-management-current.png`
   - 展示“成员管理”已经作为顶部独立入口。
   - 当前名册内展示成员、部门、小组、职位、任期和退出操作。
   - 页面不再显示“我的社团身份”页签。
3. `03-member-term-history-filter.png`
   - 展示成员任期历史入口。
   - 历史视图支持按届、部门、小组筛选，便于查看任期变化。
4. `04-member-transition-workbench.png`
   - 展示换届管理工作台。
   - 换届只处理到期待换届成员，由负责人或指导教师按目标学年继续处理。
5. `05-member-department-group-controls.png`
   - 展示当前名册中的部门和小组统计。
   - 负责人可新增部门和小组，部长可在自己部门内新增小组；成员编辑时从既定字段中选择。
6. `06-club-registration-workbench.png`
   - 展示“社团注册”作为顶部独立入口。
   - 学生可提交注册申请并查看自己的申请状态；审核权限账号可进入审核池。
7. `07-top-navigation-expanded.png`
   - 展示顶部导航完整展开。
   - “我的社团”“成员管理”“社团注册”已拆成并列入口，角色标签和退出按钮不再相互覆盖。
8. `08-club-registration-apply-dialog.png`
   - 展示社团注册申请弹窗。
   - 申请侧填写社团名称、类别、简介、申请理由、材料链接和联系方式。

## 审查重点

- “我的社团”是否只保留社团档案和个人身份概览。
- “成员管理”是否独立承载成员名册、任期历史、换届管理、部门和小组维护。
- “社团注册”是否独立承载学生申请和平台审核入口。
- 顶部导航是否完整展示，角色标签是否逐项展示且不遮挡退出和连接状态。
